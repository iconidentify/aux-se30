# Micropolis (SimCity) UI overhaul for A/UX

A ground-up modernization of the X11 Micropolis interface, built and run on
A/UX 3.1.1 (Apple's UNIX). These are the working, deployed copies of the Tcl/Tk
UI scripts (drop into `/usr/SimCity/res/`) and the new image assets (drop into
`/usr/SimCity/images/`).

The UI is entirely Tcl/Tk - the C engine (`res/sim`, see `../patched-src/`)
never changed for any of this. It runs on the bundled **Tcl 6.4 / Tk 2.3**
(early-1990s, pre-Tk-4.0), which has some sharp behavioral differences from
modern Tk that drove most of the work below (see "Old-Tk gotchas").

## What changed

### Editor
- **Custom tool palette** - a fresh, consistently themed 18-tool icon set
  (32x32, flat category-colored tile + simple white pictograph) replacing the
  large varied native art. Each tool has a base and a selected (`*hi`) variant;
  selecting brightens the tile and adds a yellow ring. Generated by
  `tools/gen_icons.py` -> PNG, then `tools/png2xpm.py` -> small-palette XPM.
- **Compact 2-column palette** in packed row-frames that hugs the button grid,
  with a left-justified **RCI demand gauge** (the original `demandg.xpm`) and a
  clickable **mini-map icon** (`mapicon.xpm`, opens the Map window) beside it,
  and a fixed-size tool name/cost **status panel** at the bottom.
- **Consolidated menu bar** - the old Controls-window menus moved into the
  editor: File, Options, Speed, Disasters, Windows, Help. Standard labels (no
  "!"), File has New City / Open City / Save / Save As / Quit.
- **Bottom status bar** - date, funds, and the notification message.
- Window title is `SimCity - <city name>`.

### Start / Welcome flow
- The unused wish root window (`SimCity Root`) is hidden.
- A revamped **Welcome screen** (`wscen.tcl`): a clean SimCity wordmark
  (`simcitybig.xpm`), an About panel with the real credits, and two actions -
  **Start New City** (opens a New City dialog: name + difficulty) and **Load
  City** (file picker). The scenario picker icons are gone; it's file-based.
- A dedicated **About SimCity** window (no longer the generic notice).
- Launch goes straight to the Welcome screen.

### Modals
- The ask/confirm dialog (`wask.tcl`) is redesigned: no colored banner, a
  centered message, uniform-width buttons, an Enter-key default with a focus
  ring, and the window is titled for its use case (e.g. "Quit SimCity").

### Other
- The Controls window is retired and repurposed as a **Messages** window (chat
  log + Talk input), reachable from Windows > Messages.
- "SimCity Overall Map" -> "Map".
- The 1993 license-**key / registration** system is removed - the game is fully
  unlocked (no Get Key menu item, no license gate).

## Depth-8 (Quadra) vs depth-1 (SE/30)

The same scripts and the same `res/sim` binary serve both displays:
- **Quadra / QEMU, 8-bit PseudoColor** - full color tiles and palette. The
  shared X colormap is the constraint: new XPMs must use few colors and reuse
  already-allocated pixels, or color allocation elsewhere starts failing. The
  new icons are 4-7 colors; the logo/gauge are <=16; that's deliberate.
- **SE/30, 1-bit StaticGray** - black & white. The engine handles the depth-1
  rendering (see `../patched-src/`); the UI scripts are depth-agnostic. The
  palette falls back to the B&W tiles automatically.

## Old-Tk gotchas (Tk 2.3, pre-bindtags) captured here

These cost the most time and are the reason the code looks the way it does:

- **An instance binding REPLACES the class binding** for that event (only the
  most specific fires; there is no `bindtags`). Putting `<Enter>`/`<Leave>`
  directly on a button overrides Tk's own `tk_butEnter`/`tk_butLeave`, which
  maintain the armed state `tk_butUp` checks before firing `-command` - so
  clicks silently stop working after the first hover. Fix: the hover bindings
  call `tk_butEnter %W` / `tk_butLeave %W` themselves before the status update.
- **Tool buttons must be PACKED** (in row frames), not `place`d - placed
  buttons drop most click events here.
- **`frame -geometry` is ignored; `frame -width`/`-height` IS honored.** Use
  the latter to pin a frame's size.
- Keep dialog/status panels a **fixed size** (frame `-width`/`-height` +
  `place`d labels) so changing text can't reflow and shift packed widgets.
- **No `raise`/`lower`, no `pack propagate`.** `pack unpack` removes a slave
  (the Tk-4.0 name is `pack forget`).
- **`wm geometry $w +x+y`** (position only) lets a toplevel auto-size to its
  content; a fixed `WxH` leaves dead space if content is smaller.
- **The Maxis `-bitmap "@file"` loader reads color XPM**, but caps colors
  (shipped art is <=8) and width (~123px). A 24-color/140px logo fails to load;
  keep new bitmaps small and few-colored.
- Removing the scenario window's Previous/Next/vote buttons broke unrelated
  code (`InitScenario`, `InitVotesForUseThisMap`, `GotoHistory`) that blindly
  configured them - which silently killed **city loading**. Guard old widget
  references with `catch`.
- Loading a city must NOT call `UINewGame` (its `sim InitGame` wipes the loaded
  city's saved graph history); initialize the windows but keep sim state.

## Files

- `simcity.tcl` - main UI logic (windows, pictures, start flow, modals, ...)
- `weditor.tcl` - the city editor window (menu bar, palette, status bar)
- `whead.tcl` - the Messages window (formerly Controls)
- `wscen.tcl` - the Welcome / start screen
- `wask.tcl` - the ask/confirm modal
- `images/ic*.xpm` - 18 tool icons + 18 selected variants
- `images/simcitybig.xpm` - the SimCity wordmark; `images/mapicon.xpm` - mini-map
- `../tools/` - the asset generators (`gen_icons.py`, `png2xpm.py`,
  `gen_mapicon.py`, `gen_simcity_logo.py`)

The UI scripts are modified GPL Micropolis (Don Hopkins / EA, GPL) UI code.
EA/Maxis demo data (sounds, tiles, cities) is not included - bring your own.
