# Micropolis (SimCity) UI modernization for A/UX

Modernized editor UI for the A/UX X11R6 build of Micropolis, deployed to
`/usr/SimCity/res/` (the `.tcl`) and `/usr/SimCity/images/` (the `.xpm`) on the
guest. These are the working, deployed copies.

## What changed

- **Custom palette icons** - a fresh, consistently themed 18-tool icon set
  (32x32, flat category-colored tile + simple white pictograph) replacing the
  large varied native art. Each tool has a base and a selected (`*hi`) variant;
  the selected variant brightens the tile and adds a yellow ring. Generated by
  `tools/gen_icons.py` -> PNG, converted to small-palette XPM by
  `tools/png2xpm.py`. Icons use few colors and reuse already-allocated pixels to
  avoid exhausting the depth-8 shared colormap.
- **Compact 2-column palette** that hugs the button grid, with an RCI demand
  gauge (`images/rcigauge.xpm`) directly beneath the tools and a fixed-size
  status panel (tool name + cost) at the bottom.
- **Consolidated menu bar** in the editor window: File, Options, Speed,
  Disasters, Windows, Help - all the old Controls-window menus moved over.
- **Bottom status bar**: date, funds, and the notification message.

## Old-Tk gotchas captured here (Tk ~3.x, pre-bindtags)

- `frame -geometry` is ignored; `frame -width`/`-height` IS honored. Use the
  latter to pin frame sizes.
- No `raise`/`lower` commands; no `pack propagate`.
- An instance binding REPLACES the widget's class binding for that event (only
  the most specific fires). Putting `<Enter>`/`<Leave>` directly on a button
  overrides `tk_butEnter`/`tk_butLeave`, which maintain the armed state
  `tk_butUp` checks before firing `-command` - so clicks silently stop working
  after the first hover. Fix: the hover bindings call `tk_butEnter %W` /
  `tk_butLeave %W` themselves before the status update (see `weditor.tcl`).
- Tool buttons must be PACKED (in row frames), not `place`d - placed buttons
  drop most click events here.
- Keep the status panel a fixed size (frame `-width`/`-height` + `place`d
  labels) so hover text changes can't reflow and shift the packed buttons.

## Files

- `weditor.tcl`, `simcity.tcl` - deployed editor UI (copy to `/usr/SimCity/res/`)
- `images/ic*.xpm` - 18 tool icons + 18 selected variants
- `images/rcigauge.xpm` - RCI demand gauge background
- `tools/gen_icons.py`, `tools/png2xpm.py` - icon generator + XPM converter
